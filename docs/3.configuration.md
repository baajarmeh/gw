# Configuration

Gw中的配置管理

## 配置分类

gw 中的配置文件，分为2种类型

1、引导级配置

所谓引导级的配置，就是在程序初始化之前（和Linux系统的引导文件类似），用于初始化必要的一些配置/设置的配置文件。

2、应用级配置

应用级的配置文件，支持的 Provider，Gw 内置了本地文件，远程 http 文件2种类型，默认使用本地文件的
Provider提供，当然也可以很容易的集成其它的应用配置，如果需要集成自定义的应用配置Provider，请参考《集成自定义配置Provider》章节的内容。

## 使用指导

1、配置 BootStrap config

2、配置 Application config

### 1、配置 BootConfig

Location: `config/boot.yaml`

```yaml
version: 1.0
configProvider: <Your provider name>
# Write your any config item at here Sections.
configuration:
  path: "config/app.yaml"

```

configProvider 内置支持

1、本地文件：`lf/local/localfs/localfile` (其中之一)

2、gw-conf-server（未来支持）：`gw-conf/gw-infra/gw-infra-conf` (其中之一)

### 自定义应用配置Provider

如果需要集成自定义的配置Provider，需要 2 个步骤

1、实现 conf.ApplicationConfigProvider 接口

```go

package myconf

type ApplicationConfigProvider interface {
	Provide(bcs BootConfig, out *ApplicationConfig) error
}
```

2、注册 Provider

```go
package myapp

import "github.com/oceanho/gw"

func init(){
    gw.RegisterConfigProvider(<Your provider>)
}
```

3、使用自定义的 Provider

```yaml
version: 1.0
configProvider: <Your provider name>
# Write your any config item at here Sections.
configuration:
  path: "config/app.yaml"

```
